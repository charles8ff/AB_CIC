Explain the XGBoost model with SHAP values

Because the Tree SHAP algorithm is implemented in XGBoost we can compute exact SHAP values quickly over thousands of samples. The SHAP values for a single prediction (including the expected output in the last column) sum to the model's output for that prediction.

In[]:
# Compute the SHAP values for every prediction in the validation dataset
explainer = shap.TreeExplainer(model)
shap_values = explainer.shap_values(Xv)

Explain a single player's chances of winning a particular match

The sum of SHAP values is the difference between the model's expected output and the current output for the player. It should be noted that the Tree SHAP implementation explains the model's margin output rather than the transformed output (for example, a logistic regression probability). This means that the SHAP values in this model are expressed in log odds ratios. A large positive value indicates that a player is likely to win, whereas a large negative value indicates that they are more likely to lose.

Summarize the impact of all features over the entire dataset

A SHAP value for a feature in a specific prediction indicates how much the model prediction changes when that feature is observed. In the summary plot below, we show all of the SHAP values for a single feature (such as Gold Earned per min) in a row, with the x-axis representing the SHAP value. By repeating this for all features, we can determine which features have a significant impact on the model's prediction (such as Gold Earned per min) and which have a minor impact (such as Total healed units per min). Note that when points do not fit together on the line, they pile up vertically to demonstrate density. Each dot is additionally coloured by the value of that feature from high to low.

Examine how changes in a single feature change the model's prediction

The XGBoost model we trained above is extremely complex, but by plotting the SHAP value for a feature against its actual value for all players, we can see how changes in the feature's value affect the model's output. These plots are very similar to standard partial dependence plots, but they also show how much context matters for a feature (or how much interaction terms matter). The vertical dispersion of data points reflects how much interaction terms influence the importance of a feature. For example, earning only 100 gold per minute during a game may reduce your logg odds of winning by 10 for some players and only 3 for others. This occurs because other features of these players effect how much earning gold matters for winning the game. Note that the vertical spread narrows once you earn at least 500 gold/min, meaning the context of other features matters less for high gold earners than low gold earners. We color the datapoints with another feature that most explains the interaction effect variance. For example earning less gold is less bad if you have not died very much, but it is really bad if you also die a lot.

The y-axis in the plots below represents the SHAP value for that feature, so -4 means observing that feature lowers your log odds of winning by 4, while a value of +2 means observing that feature raises your log odds of winning by 2.

Note that these plot just explain how the XGBoost model works, not necessarily how reality works. Since the XGBoost model is trained from observational data, it is not nessecarily a causal model, and so just because changing a factor makes the model's prediction of winning go up, does not always mean it will raise your actual chances.
